# QUIZ FIX DEPLOYMENT GUIDE
**Date:** April 11, 2026  
**Issue:** Quiz buttons not responding on live site  
**Root Cause:** Wrong quiz.js version deployed with broken inline onclick handlers

---

## DIAGNOSIS

**Live site analysis revealed:**
- Homepage loads correctly ✅
- CSS loads correctly ✅
- quiz.js file accessible BUT contains OLD CODE ❌

**Bug confirmed at line 2:**
```javascript
// ExpatAce Quiz Engine
// 12 questions, city-level scoring, dual entry points  ← WRONG VERSION
```

**Should say:**
```javascript
// ExpatAce Quiz Engine
// 16 questions, city-level scoring, dual entry points  ← CORRECT VERSION
```

**Technical bug:** Inline onclick handlers like `onclick="selectAnswer('${opt.value}', this)"` cause `this` keyword to refer to window object instead of button element, breaking selection logic.

---

## THE FIX

**Changed from (BROKEN):**
```javascript
<button class="option-btn" onclick="selectAnswer('${opt.value}', this)">
  ...
</button>
```

**Changed to (FIXED):**
```javascript
// Step 1: Button markup with data attribute
<button class="option-btn" data-value="${opt.value}">
  ...
</button>

// Step 2: Event delegation AFTER innerHTML rendering
document.querySelectorAll('.option-btn').forEach(btn => {
  btn.addEventListener('click', function() {
    selectAnswer(this.dataset.value, this);
  });
});
```

**Now:** `this` correctly refers to clicked button element → selection works perfectly.
